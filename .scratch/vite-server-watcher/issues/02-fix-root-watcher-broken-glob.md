# Fix rootWatcher: chokidar v5 dropped glob support, so PDF add/unlink detection is broken

## Context

`packages/note-first-presenter/src/vite/plugin.ts` の `createSlidesContext` 内 `rootWatcher` は、cwd への PDF 追加・削除を検知するために glob を使っている:

```ts
const rootWatcher = chokidar
  .watch('*.pdf', { depth: 0, ignoreInitial: true })
  .on('add', trigger)
  .on('unlink', trigger);
```

chokidar は v4 で glob サポートを削除した。本パッケージは chokidar `^5.0.0` に依存しているため、`'*.pdf'` はリテラルなファイル名として扱われ、**PDF を cwd に追加・削除しても add/unlink イベントは一切発火しない**(2026-06-10 に実験で確認済み)。起動時に存在した PDF は初期 `reload()` の `resolveSlides`(tinyglobby 使用)で拾われるため、起動後の追加・削除だけが静かに失われる。

他の 2 つの watcher は chokidar v5 でも正常に機能していることを検証済み:

- `configWatcher` — リテラルパス監視。まだ存在しない設定ファイルが後から作られるケースも検知できる
- `dynamicWatcher` — 絶対パスのリテラル監視

## Fix

glob をやめ、cwd ディレクトリ自体を `depth: 0` で監視し、イベントハンドラの先頭で `.pdf` 以外を除外する:

```ts
const rootWatcher = chokidar
  .watch('.', { depth: 0, ignoreInitial: true })
  .on('add', onRootEvent)
  .on('unlink', onRootEvent);
// onRootEvent: if (!p.endsWith('.pdf')) return; trigger();
```

- イベントのパスは相対で届く(検証済み)
- `.pdf` フィルタは必須。glob が機能していた頃と違い、ディレクトリ監視では cwd 直下のあらゆるファイルのイベントが届くため、フィルタがないと無関係なファイル(エディタの一時ファイル、db ファイルなど)の作成のたびに reload と full-reload が走る
- フィルタの置き場所は `ignored` オプションではなくハンドラとする。cwd 直下という小さなスコープでは資源面の差は誤差であり、ハンドラ先頭の 1 行のほうが意図が読み取りやすい
