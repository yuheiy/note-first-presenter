# セパレータはハイフン3個以上（CommonMark thematic break の絞り込み版）

ノートグループ（=スライド）の区切りは、テキストがちょうど `---` ではなく、**ハイフンが3個以上連続**（`/^-{3,}$/`）したトップレベル `list_item` で表す。ユーザーが `----` や `-----` のように個数を間違えても区切りとして機能させるための緩和である。CommonMark の thematic break を参考にしつつ、意図的に次の点を絞り込む。

- 対応文字は `-`（ハイフン）のみ。`*` / `_` は対象外。
- 連続したハイフンのみ。`- - -` のようなスペース混在は対象外。
- 前後の空白や他文字を一切含まない。`  ---` や `--- foo` は区切りにならない。

## Considered Options

- **CommonMark thematic break のフル互換**（`*` / `_`、スペース混在も許容）: 却下。アウトライナーは段落がプレーンテキストであり、`***` のような強調記号の打ち間違いや、箇条書きとして書いた `- - -` を意図せず区切り化してノートを失わせるリスクがある。動機は「ハイフンの個数の間違いの許容」に限られるため、ハイフンのみ・連続・3個以上に絞れば十分。

## Consequences

- 判定は client（ProseMirror Node の `isTopLevelSeparator`）と cli / client（JSON Doc の `isSeparatorItem`）で独立実装のまま、同一の `/^-{3,}$/` ルールを各々が持つ（[ADR 0003](./0003-separator-is-a-runtime-rule.md) の方針を維持）。
- 構造条件（トップレベルの `list_item` で子は `paragraph` 1つのみ、ネストした子バレットを持たない）は据え置き。
