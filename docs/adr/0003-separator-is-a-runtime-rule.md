# セパレータは専用ノード型ではなく `---` の runtime ルール

ノートグループ（=スライド）の区切りは、テキストがちょうど `---` のトップレベル `list_item` で表す。これを専用の ProseMirror ノード型にはせず、通常の `list_item` に対する runtime 述語（`isTopLevelSeparator` / `isSeparatorItem`）として判定する。したがってセパレータの編集・移動・削除・複製はすべて通常のバレットと同一に振る舞い、「セパレータか否か」は常にテキストから再計算される。

## Considered Options

- **専用セパレータノード型**: 却下。全コマンドとスキーマに特別扱いを波及させる必要があり、プレーンテキスト限定の素朴なスキーマも崩れる。通常バレットとして扱えば編集系コマンドがそのまま動く。

## Consequences

- セパレータ判定は client（ProseMirror Node）と cli（JSON Doc）の2つの独立実装を持ち、各々が独立にテストされる。
